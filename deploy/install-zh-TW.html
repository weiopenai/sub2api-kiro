使用 Docker 與 GitHub Container Registry 映像檔在全新伺服器上部署

1. 環境需求

  • Linux 伺服器（Ubuntu / Debian / CentOS 等）
  • 已安裝 Docker 與 Docker Compose v2
  • 至少 2GB RAM（建議 4GB 以上）
  • 對外開放的連接埠（本指南使用 8880）

映像檔已公開發布於：ghcr.io/weiopenai/sub2api-kiro:latest

2. 安裝 Docker（若尚未安裝）

# 使用官方一鍵安裝腳本
curl -fsSL https://get.docker.com | sh

# 啟動並設為開機自動啟動
sudo systemctl enable --now docker

# 將目前使用者加入 docker 群組（免 sudo，需重新登入生效）
sudo usermod -aG docker $USER

確認版本：

docker --version
docker compose version

3. 準備部署檔案

建立部署目錄，並下載 docker-compose.yml 與 .env 範例：

mkdir -p ~/sub2api-kiro && cd ~/sub2api-kiro

# 從專案倉庫取得部署檔（或自行用 scp 傳上來）
curl -O https://raw.githubusercontent.com/weiopenai/sub2api-kiro/main/deploy/docker-compose.yml
curl -o .env https://raw.githubusercontent.com/weiopenai/sub2api-kiro/main/deploy/.env.example
若倉庫為私有或無法直接下載，請從你本機的 deploy/ 目錄把 docker-compose.yml 與 .env 用 scp 傳到伺服器。映像檔本身已公開，docker compose up 會自動拉取。

4. 設定環境變數

編輯 .env，務必修改以下幾項：

nano .env
變數說明建議值
SERVER_PORT對外連接埠（避免用常見的 8080）8880
POSTGRES_PASSWORD資料庫密碼（必改）openssl rand -hex 16 產生
JWT_SECRET登入 token 簽章金鑰（固定值避免重啟後登出）openssl rand -hex 32 產生
TOTP_ENCRYPTION_KEY2FA 加密金鑰（固定值避免 2FA 失效）openssl rand -hex 32 產生
ADMIN_EMAIL管理員帳號你的 Email
ADMIN_PASSWORD管理員密碼（留空則自動生成並印在 log）留空或自訂
TZ時區Asia/Taipei

快速產生隨機金鑰：

echo "JWT_SECRET=$(openssl rand -hex 32)"
echo "TOTP_ENCRYPTION_KEY=$(openssl rand -hex 32)"
echo "POSTGRES_PASSWORD=$(openssl rand -hex 16)"
重要：JWT_SECRET 與 TOTP_ENCRYPTION_KEY 一定要設成固定值。如果留空，每次容器重啟都會重新生成，導致所有使用者被登出、2FA 失效。

5. 啟動服務

# 確認 docker-compose.yml 的 image 為：
#   image: ghcr.io/weiopenai/sub2api-kiro:latest

# 拉取映像檔並在背景啟動
docker compose up -d

# 查看啟動日誌
docker compose logs -f sub2api

確認三個容器都是 healthy：

docker ps --filter "name=sub2api"
看到 sub2api、sub2api-postgres、sub2api-redis 都顯示 (healthy) 就代表啟動成功。

6. 取得管理員密碼並登入

如果 .env 的 ADMIN_PASSWORD 留空，系統會在首次啟動時自動生成密碼，從 log 取得：

docker logs sub2api 2>&1 | grep -i "Generated admin password"

接著用瀏覽器開啟（把 SERVER_IP 換成伺服器 IP）：

http://SERVER_IP:8880

使用 .env 設定的 ADMIN_EMAIL 與上面取得的密碼登入。

登入後第一件事：到右上角語言切換器選擇 🇹🇼 繁體中文，並盡快修改管理員密碼。

7. 更新與維護

更新到最新映像檔

cd ~/sub2api-kiro
docker compose pull sub2api
docker compose up -d --no-deps sub2api

重啟服務（不更新）

docker compose restart sub2api

查看日誌

docker compose logs -f sub2api
切勿使用 docker compose down -v —— 參數 -v 會刪除資料庫 volume，導致所有資料遺失。單純停止服務請用 docker compose down（不加 -v）。

8. 常見問題

連接埠被佔用

修改 .env 的 SERVER_PORT 後，重建容器：

docker compose down
docker compose up -d

無法拉取映像檔

確認映像檔名稱正確且為公開：ghcr.io/weiopenai/sub2api-kiro:latest。若改為私有，需先登入：

echo $GITHUB_TOKEN | docker login ghcr.io -u weiopenai --password-stdin

資料庫資料保留

資料存放於 Docker named volume（postgres_data），只要不執行 down -v，更新或重啟都不會遺失資料。

備份資料庫

docker exec sub2api-postgres pg_dump -U sub2api sub2api > backup_$(date +%F).sql
